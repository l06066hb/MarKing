## 🐛 问题反馈

### Bug 报告
如果您遇到了软件缺陷或错误，请提交 Bug 报告：

➡️ **[提交 Bug 报告](https://github.com/l06066hb/MarKing/issues/new?template=bug_report.md)**

在报告中请包含：
- 详细的问题描述
- 复现步骤
- 系统信息（操作系统版本、MarKing 版本）
- 错误截图或日志（如有）

### 功能建议
如果您有新功能想法或改进建议：

➡️ **[提交功能建议](https://github.com/l06066hb/MarKing/issues/new?template=feature_request.md)**

请描述：
- 功能需求和使用场景
- 期望的实现方式
- 如何帮助您提升工作效率

### 使用咨询
如果您有使用问题或需要帮助：

➡️ **[提问咨询](https://github.com/l06066hb/MarKing/issues/new?template=question.md)**

## 📖 自助解决

在提交问题前，建议先尝试以下资源：

### 1. 查看文档
- 浏览 [常见问题 (FAQ)](docs/FAQ.md)
- 查看 [故障排除指南](docs/TROUBLESHOOTING.md)
- 阅读 [更新日志](CHANGELOG.md) 确认问题是否已修复

### 2. 搜索已有 Issue
在提交新问题前，请先搜索 [现有 Issues](https://github.com/l06066hb/MarKing/issues)，避免重复报告。

### 3. 应用内帮助
- 按 `?` 键查看快捷键帮助
- 查看设置面板中的说明文本
- 尝试不同的配置选项

## 🔄 版本更新

### 检查更新
MarKing 支持自动检查更新：
1. 打开设置（齿轮图标）
2. 切换到"关于"标签
3. 点击"检查更新"

### 下载新版本
访问 [Releases 页面](https://github.com/l06066hb/MarKing/releases) 下载最新版本。

## 📊 问题处理流程

提交问题后，我们的处理流程：

1. **问题分类** - 我们会为问题添加标签（bug/enhancement/question 等）
2. **优先级评估** - 根据影响范围和严重程度设置优先级
3. **问题跟踪** - 您可以在 Issue 中查看处理进度
4. **版本修复** - 修复后的功能会在更新日志中说明
5. **Issue 关闭** - 问题解决后关闭，欢迎验证和反馈

## 💡 最佳实践

为了获得更快更好的帮助，建议：

✅ **提供详细信息** - 详细的描述有助于快速定位问题  
✅ **使用问题模板** - 模板确保包含必要信息  
✅ **一个 Issue 一个问题** - 不要在同一个 Issue 中混合多个不相关的问题  
✅ **保持礼貌** - 友好的交流氛围让协作更顺畅  
✅ **及时反馈** - 如果问题解决或有新进展，请更新 Issue
